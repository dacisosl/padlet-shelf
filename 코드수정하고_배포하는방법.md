# 📝 코드 수정하고 배포하는 방법 (초보자용)

## 🎯 전체 흐름

1. 로컬에서 코드 수정
2. GitHub Desktop에서 변경사항 확인
3. 커밋 (변경사항 저장)
4. Push (GitHub에 올리기)
5. Netlify가 자동으로 배포

---

## 1️⃣ 단계: 코드 수정하기

### 예시: 카드 삭제 버튼 수정

1. **코드 에디터에서 파일 수정**
   - 예: `src/components/Card.jsx` 파일 수정
   - 저장 (Ctrl + S)

2. **수정 완료!**
   - 이제 GitHub Desktop으로 이동

---

## 2️⃣ 단계: GitHub Desktop 열기

1. **GitHub Desktop 실행**
   - 바탕화면 아이콘 더블클릭
   - 또는 시작 메뉴에서 "GitHub Desktop" 검색

2. **프로젝트 확인**
   - 좌측 상단에 "Current repository: padlet-shelf" 표시 확인
   - 다른 프로젝트면 드롭다운에서 `padlet-shelf` 선택

---

## 3️⃣ 단계: 변경사항 확인하기

1. **좌측 하단 "Changes" 탭 확인**
   - 수정한 파일 목록이 보임
   - 예: `Card.jsx` 파일이 목록에 표시됨

2. **변경 내용 확인**
   - 파일을 클릭하면 수정된 내용이 보임
   - 초록색 = 추가된 내용
   - 빨간색 = 삭제된 내용

---

## 4️⃣ 단계: 커밋하기 (변경사항 저장)

1. **좌측 하단 "Summary" 입력란에 메시지 입력**
   - 예: `카드 삭제 버튼 항상 표시`
   - 또는: `UI 개선`
   - 또는: `버그 수정`

2. **"Commit to main" 버튼 클릭**
   - 좌측 하단에 있는 버튼
   - 클릭하면 변경사항이 로컬에 저장됨

3. **완료 확인**
   - "Changes" 탭에 파일이 사라지면 성공!
   - "History" 탭에 새 커밋이 추가됨

---

## 5️⃣ 단계: GitHub에 올리기 (Push)

1. **상단 메뉴 확인**
   - "Push origin" 버튼이 보임
   - 또는 상단에 "1 commit ahead of origin/main" 메시지가 보임

2. **"Push origin" 버튼 클릭**
   - 상단 중앙 또는 우측 상단에 있는 버튼
   - 클릭하면 GitHub에 업로드 시작

3. **업로드 완료 확인**
   - "Pushed to origin/main" 메시지가 나오면 성공!
   - 또는 "Up to date with origin/main" 메시지

---

## 6️⃣ 단계: Netlify 자동 배포 확인

1. **Netlify 대시보드 접속**
   - 브라우저에서 `app.netlify.com` 접속
   - 로그인

2. **프로젝트 선택**
   - `padlet-shelf` 또는 `padllet` 프로젝트 클릭

3. **Deploys 탭 확인**
   - 새로운 배포가 자동으로 시작됨
   - "Building" 또는 "Deploy in progress" 표시

4. **배포 완료 대기**
   - 약 1-2분 소요
   - 초록색 체크(✓) 표시가 나오면 완료!

5. **사이트 확인**
   - 사이트 URL로 접속하여 변경사항 확인

---

## 📸 단계별 스크린샷 설명

### GitHub Desktop 화면 구성

```
┌─────────────────────────────────────┐
│  [File] [Edit] [View] [Repository]  │ ← 상단 메뉴
├─────────────────────────────────────┤
│                                     │
│  Current repository: padlet-shelf   │ ← 현재 프로젝트
│                                     │
│  ┌─────────────────────────────┐   │
│  │ Changes (2)                 │   │ ← 변경된 파일 수
│  │                              │   │
│  │ 📄 Card.jsx                  │   │ ← 수정한 파일
│  │    Modified                  │   │
│  │                              │   │
│  │ 📄 index.css                 │   │
│  │    Modified                  │   │
│  └─────────────────────────────┘   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ Summary                      │   │ ← 커밋 메시지 입력
│  │ [카드 삭제 버튼 수정]         │   │
│  │                              │   │
│  │ Description (optional)       │   │
│  │                              │   │
│  │ [Commit to main]             │   │ ← 커밋 버튼
│  └─────────────────────────────┘   │
│                                     │
│  [Push origin]                      │ ← 푸시 버튼
└─────────────────────────────────────┘
```

---

## 💡 자주 하는 실수와 해결법

### 실수 1: "Commit to main"만 하고 Push 안 함
**증상**: GitHub에 올라가지 않음
**해결**: "Push origin" 버튼 클릭!

### 실수 2: 커밋 메시지를 입력 안 함
**증상**: "Commit to main" 버튼이 비활성화됨
**해결**: Summary에 메시지 입력

### 실수 3: 잘못된 프로젝트 선택
**증상**: 변경사항이 안 보임
**해결**: 좌측 상단에서 `padlet-shelf` 선택

- [ ] 변경사항이 "Changes" 탭에 보임
- [ ] Summary에 커밋 메시지 입력
- [ ] "Commit to main" 클릭
- [ ] "Push origin" 클릭
- [ ] Netlify에서 배포 진행 확인
- [ ] 사이트에서 변경사항 확인
