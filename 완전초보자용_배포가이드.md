## 1️⃣ 단계: GitHub 계정 만들기 (계정이 없으면)

### 1-1. GitHub 웹사이트 접속
1. 웹 브라우저 열기 (Chrome, Edge 등)
2. 주소창에 `github.com` 입력하고 Enter
3. 우측 상단 **Sign up** 클릭

### 1-2. 계정 정보 입력
1. **Username**: 원하는 사용자 이름 입력 (예: `myusername`)
2. **Email**: 이메일 주소 입력
3. **Password**: 비밀번호 입력 (8자 이상)
4. **Verify your account**: 로봇이 아닌지 확인하는 체크박스 클릭
5. **Create account** 클릭

### 1-3. 이메일 인증
1. 입력한 이메일로 가서 GitHub에서 온 메일 확인
2. 메일 안의 링크 클릭하여 인증 완료

## 3️⃣ 단계: Firebase 설정 확인 (중요!)

배포 전에 Firebase 설정이 완료되어 있어야 합니다.

### 3-1. Firebase 콘솔 접속
1. 브라우저에서 `console.firebase.google.com` 접속
2. 프로젝트 `test-ec573` 선택

### 3-2. Authentication 확인
1. 왼쪽 메뉴에서 **Authentication** 클릭
2. **Sign-in method** 탭 클릭
3. **익명** 로그인 방법 찾기
4. **사용 설정** 토글이 **ON**인지 확인
   - OFF면 클릭해서 ON으로 변경
   - **저장** 클릭

### 3-3. Firestore Database 확인
1. 왼쪽 메뉴에서 **Firestore Database** 클릭
2. 데이터베이스가 생성되어 있는지 확인
   - 없으면 **데이터베이스 만들기** 클릭
   - **테스트 모드에서 시작** 선택
   - 위치: **asia-northeast3 (서울)** 선택
   - **사용 설정** 클릭

3. **Rules** 탭 클릭하여 다음 규칙 확인:
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /columns/{document=**} {
      allow read, write: if true;
    }
    match /cards/{document=**} {
      allow read, write: if true;
    }
  }
}
```
- 다르면 복사해서 붙여넣기
- **게시** 버튼 클릭

### 3-4. Storage 확인
1. 왼쪽 메뉴에서 **Storage** 클릭
2. Storage가 생성되어 있는지 확인
   - 없으면 **시작하기** 클릭
   - **Firestore의 보안 규칙 사용** 선택
   - 위치: **asia-northeast3 (서울)** 선택
   - **완료** 클릭

3. **Rules** 탭 클릭하여 다음 규칙 확인:
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /cards/{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```
- 다르면 복사해서 붙여넣기
- **게시** 버튼 클릭

✅ **Firebase 설정 완료!**

## 📝 자동 배포 설정 (이미 완료됨!)

앞으로 코드를 수정하면:
1. GitHub에 코드 푸시
2. Netlify가 자동으로 감지
3. 자동으로 재배포 시작
4. 몇 분 후 새 버전이 사이트에 반영됨

**수동으로 재배포하려면:**
- Netlify 대시보드 → **Deploys** 탭
- **Trigger deploy** → **Clear cache and deploy site** 클릭

## ❓ 자주 묻는 질문 (FAQ)

### Q1. 배포가 실패했어요
**A:** Netlify 대시보드 → **Deploys** → 실패한 배포 클릭 → 로그 확인
- 환경 변수가 모두 추가되었는지 확인
- 빌드 명령어가 올바른지 확인

### Q2. 사이트는 열리는데 Firebase 연결이 안 돼요
**A:** 
1. Firebase 콘솔 → Authentication → Settings → Authorized domains 확인
2. Netlify 도메인이 추가되었는지 확인
3. 환경 변수가 모두 올바르게 입력되었는지 확인

### Q3. 코드를 수정했는데 사이트에 반영이 안 돼요
**A:**
1. GitHub에 푸시했는지 확인
2. Netlify 대시보드에서 배포가 진행 중인지 확인
3. 수동 재배포: **Deploys** → **Trigger deploy**

### Q4. 도메인을 바꾸고 싶어요
**A:**
- Netlify 대시보드 → **Domain settings** → **Options** → **Edit site name**
- 원하는 이름 입력 → **Save**
