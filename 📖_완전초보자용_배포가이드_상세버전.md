# 🎯 PART 1: GitHub 계정 만들기

## 1-1. GitHub 웹사이트 접속

1. **웹 브라우저 열기**
   - Chrome, Edge, Firefox 등 아무 브라우저나 사용 가능

2. **주소창에 입력**
   ```
   github.com
   ```
   Enter 키 누르기

3. **화면 확인**
   - GitHub 홈페이지가 열림
   - 우측 상단에 **Sign up** 버튼이 보임

## 1-2. 계정 만들기

1. **Sign up 버튼 클릭**

2. **정보 입력하기**
   - **Username**: 원하는 사용자 이름 입력
     - 예: `myusername`, `johndoe` 등
     - 영어, 숫자, 하이픈(-)만 사용 가능
     - 중복 확인: 초록색 체크 표시가 나오면 사용 가능
   
   - **Email address**: 이메일 주소 입력
     - 예: `myemail@gmail.com`
     - 실제로 사용하는 이메일이어야 함 (인증 필요)
   
   - **Password**: 비밀번호 입력
     - 최소 8자 이상
     - 숫자, 영문자 포함 권장
     - 입력하면 강도 표시가 나타남

3. **로봇 확인**
   - "Verify you're not a robot" 체크박스 클릭
   - 퍼즐이나 이미지 선택이 나올 수 있음
   - 완료하면 체크 표시

4. **Create account 버튼 클릭**

## 1-3. 이메일 인증

1. **이메일 확인**
   - 입력한 이메일 주소로 가기
   - GitHub에서 온 메일 찾기
   - 제목: "Verify your email address" 또는 "GitHub 이메일 인증"

2. **인증 링크 클릭**
   - 메일 안에 있는 링크 클릭
   - 또는 메일 안의 코드를 복사해서 GitHub에 입력

3. **인증 완료 확인**
   - "Your email was verified" 메시지가 나오면 완료!

✅ **GitHub 계정 만들기 완료!**

# 🎯 PART 2: GitHub에 코드 올리기

## 방법 선택

두 가지 방법이 있습니다:
- **방법 A**: GitHub Desktop 사용 (⭐ 추천! 가장 쉬움)
- **방법 B**: 웹사이트 + 명령어 사용

---

## 📦 방법 A: GitHub Desktop 사용 (가장 쉬움!)

### A-1. GitHub Desktop 다운로드

1. **브라우저에서 접속**
   ```
   desktop.github.com
   ```
   Enter 키

2. **다운로드 버튼 클릭**
   - **Download for Windows** 버튼 클릭
   - 자동으로 다운로드 시작됨

3. **설치하기**
   - 다운로드된 파일 더블클릭
   - 설치 창이 뜨면 **Next** → **Install** → **Finish**
   - 설치 완료!

### A-2. GitHub Desktop 실행 및 로그인

1. **GitHub Desktop 실행**
   - 바탕화면 아이콘 더블클릭
   - 또는 시작 메뉴에서 "GitHub Desktop" 검색

2. **로그인하기**
   - 첫 실행 시 "Sign in to GitHub.com" 화면이 나옴
   - **Sign in to GitHub.com** 버튼 클릭

3. **브라우저에서 인증**
   - 브라우저가 자동으로 열림
   - **Authorize GitHub Desktop** 버튼 클릭
   - GitHub Desktop으로 돌아가면 로그인 완료!

### A-3. 프로젝트 폴더 추가

1. **GitHub Desktop에서**
   - 상단 메뉴 **File** 클릭
   - **Add Local Repository** 클릭

2. **폴더 선택**
   - **Choose...** 버튼 클릭
   - 파일 탐색기 창이 열림
   - 다음 경로로 이동:
     ```
     C:\Users\dydy7\Desktop\padlet-shelf
     ```
   - `padlet-shelf` 폴더 선택
   - **폴더 선택** 버튼 클릭

3. **저장소 추가 확인**
   - "This directory does not appear to be a Git repository" 메시지가 나올 수 있음
   - **create a repository** 클릭
   - 또는 좌측 하단에 파일 목록이 보이면 성공!

### A-4. 첫 번째 커밋 만들기

1. **변경사항 확인**
   - 좌측 하단에 파일 목록이 보임
   - 모든 파일이 체크되어 있는지 확인

2. **커밋 메시지 입력**
   - 좌측 하단 **Summary** 입력란에:
     ```
     첫 번째 커밋
     ```
     입력

3. **커밋하기**
   - **Commit to main** 버튼 클릭
   - 잠시 기다리면 완료!

### A-5. GitHub에 올리기 (Publish)

1. **Publish 버튼 클릭**
   - 상단 메뉴 **Repository** → **Publish repository** 클릭
   - 또는 좌측 상단에 **Publish repository** 버튼이 보이면 클릭

2. **저장소 정보 확인**
   - **Name**: `padlet-shelf` (그대로 두기)
   - **Description**: (선택사항) "패들렛 스타일 칸반 보드"
   - ⚠️ **Keep this code private** 체크박스 해제
     - 체크 해제 = 공개 저장소 (누구나 볼 수 있음)
     - 체크 = 비공개 저장소 (본인만 볼 수 있음)

3. **Publish 클릭**
   - **Publish repository** 버튼 클릭
   - 잠시 기다리면 완료!

4. **완료 확인**
   - "Repository published" 메시지가 나오면 성공!
   - **View on GitHub** 버튼 클릭하면 웹사이트에서 확인 가능

✅ **GitHub에 코드 올리기 완료!**

## 💻 방법 B: 웹사이트 + 명령어 사용

### B-1. GitHub에서 새 저장소 만들기

1. **GitHub.com 접속 및 로그인**
   - `github.com` 접속
   - 우측 상단 **Sign in** 클릭하여 로그인

2. **새 저장소 만들기**

3. **저장소 정보 입력**
   - **Repository name**: `padlet-shelf` 입력
   - **Description**: (선택사항) "패들렛 스타일 칸반 보드"
   - **Public** 선택 (공개 저장소)
   - ⚠️ **중요**: 아래 체크박스들은 모두 체크 해제!

     - ❌ Add .gitignore
     - ❌ Choose a license
   - **Create repository** 버튼 클릭

4. **저장소 생성 완료**
   - "Quick setup" 페이지가 나타남
   - 이 페이지는 나중에 필요하니 열어두기

3. **명령 프롬프트 실행**
   - **명령 프롬프트** 클릭
   - 검은색 창이 열림

### B-3. 명령어 입력하기 (하나씩!)

⚠️ **주의**: 각 명령어를 입력한 후 **Enter 키를 반드시 누르세요!**


- 결과: "Enumerating objects..." → "Writing objects..." → "remote: Resolving deltas..." 메시지가 나오면 성공!

## 3-3. Firestore Database 설정 확인

### 3-3-1. Firestore Database 메뉴 열기
1. 왼쪽 메뉴에서 **Firestore Database** 클릭

### 3-3-2. 데이터베이스 생성 (없으면)
1. **데이터베이스 만들기** 버튼 클릭
2. **테스트 모드에서 시작** 선택
   - ⚠️ 개발 중이므로 테스트 모드 사용
3. **다음** 클릭
4. **위치 선택**
   - 드롭다운에서 **asia-northeast3 (서울)** 선택
   - 가장 가까운 위치
5. **사용 설정** 버튼 클릭
6. 잠시 기다리면 데이터베이스 생성 완료!

### 3-3-3. Firestore 보안 규칙 설정
1. 상단 탭에서 **Rules** 클릭
2. 기존 규칙을 모두 삭제하고 다음 코드를 복사해서 붙여넣기:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /columns/{document=**} {
      allow read, write: if true;
    }
    match /cards/{document=**} {
      allow read, write: if true;
    }
  }
}
```

3. **게시** 버튼 클릭
4. ✅ "규칙이 성공적으로 게시되었습니다" 메시지 확인

## 3-4. Storage 설정 확인

### 3-4-1. Storage 메뉴 열기
1. 왼쪽 메뉴에서 **Storage** 클릭

### 3-4-2. Storage 생성 (없으면)
1. **시작하기** 버튼 클릭
2. **Firestore의 보안 규칙 사용** 선택
3. **다음** 클릭
4. **위치 선택**
   - Firestore와 동일하게 **asia-northeast3 (서울)** 선택
5. **완료** 버튼 클릭
6. 잠시 기다리면 Storage 생성 완료!

### 3-4-3. Storage 보안 규칙 설정
1. 상단 탭에서 **Rules** 클릭
2. 기존 규칙을 모두 삭제하고 다음 코드를 복사해서 붙여넣기:

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /cards/{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```

3. **게시** 버튼 클릭
4. ✅ "규칙이 성공적으로 게시되었습니다" 메시지 확인

✅ **Firebase 설정 완료!**

# ❓ 자주 묻는 질문 (FAQ)

## Q1. 배포가 실패했어요. 어떻게 해야 하나요?

**A:** 
1. Netlify 대시보드 → **Deploys** 탭 클릭
2. 실패한 배포(빨간색 X 표시) 클릭
3. **Deploy log** 확인
4. 에러 메시지 확인:
   - "Environment variable not found" → 환경 변수 확인
   - "Build command failed" → 빌드 명령어 확인
   - "Module not found" → package.json 확인

## Q2. 사이트는 열리는데 Firebase 연결이 안 돼요

**A:**
1. Firebase 콘솔 → Authentication → Settings → Authorized domains 확인
2. Netlify 도메인이 목록에 있는지 확인
3. Netlify → Site settings → Environment variables 확인
4. 모든 `VITE_`로 시작하는 변수가 있는지 확인

## Q3. 코드를 수정했는데 사이트에 반영이 안 돼요

**A:**
1. GitHub에 푸시했는지 확인
   - GitHub Desktop: Push origin 클릭했는지 확인
   - 명령어: `git push` 실행했는지 확인
2. Netlify 대시보드에서 배포가 진행 중인지 확인
3. 수동 재배포:
   - Netlify → **Deploys** 탭
   - **Trigger deploy** → **Clear cache and deploy site** 클릭

## Q4. 사이트 URL을 바꾸고 싶어요

**A:**
1. Netlify 대시보드 → 사이트 선택
2. **Domain settings** 클릭
3. **Options** → **Edit site name** 클릭
4. 원하는 이름 입력 (예: `my-padlet-board`)
5. **Save** 클릭
6. 새 URL: `https://my-padlet-board.netlify.app`

## Q5. 비공개 저장소로 바꾸고 싶어요

**A:**
1. GitHub.com 접속
2. `padlet-shelf` 저장소 열기
3. **Settings** 탭 클릭
4. 아래로 스크롤하여 **Danger Zone** 찾기
5. **Change visibility** → **Make private** 클릭
6. 확인 메시지에 저장소 이름 입력
7. **I understand, change repository visibility** 클릭
